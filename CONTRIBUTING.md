# 内容与代码贡献说明

TinyWall 当前由个人维护。修改建议请通过功能分支和 Pull Request 提交，以便使用 Cloudflare Pages 的预览地址核对结果。

## 内容规则

- 一篇文章对应一个 Markdown 或 MDX 文件。
- 不修改已经发布文章的 `slug`；如确需修改，应同步增加重定向。
- `description` 应能独立说明文章主题，不使用“点击查看”等空泛文字。
- 主题必须来自 `src/data/topics.ts`；新增主题时同时补充名称和简介。
- AI 生成内容必须保留可核验来源，事实、来源观点和作者推断应清楚区分。
- 示例、草稿和真实发布内容必须明确标注，避免示例信息被误认为事实。

## 提交前

运行 `npm run build`，确认类型检查、内容校验、静态构建和搜索索引全部成功。
